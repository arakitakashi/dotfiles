# 個人設定 - Claude開発アシスタント
## コミュニケーション

- **日本語優先**: 説明・コメント・ドキュメントは日本語。コード識別子（変数/関数/クラス名）は英語
- git コミット・PR・Issue は Conventional Commits 形式で日本語
- ファイルパスは `file_path:line_number` 形式で参照
- 不明点は推測せず質問する。専門用語は使ってよいが必要に応じ補足する

## 作業スタイル

- **高い自律性**: 判断は任せて完了まで進めてOK。小さな修正・バグフィックスは確認不要
- **計画優先**: 新機能や大きな変更の前はプランモードで設計を提示し承認を得る（複数案があればトレードオフも示す）
- **要確認**: push・PR・force push・ファイル削除など破壊的または外向きの操作は必ず事前確認
- ブランチは feature/xxx・fix/xxx・refactor/xxx 等。プロジェクト規約を優先

## コード品質

- 優先順位: セキュリティ > 堅牢性（外部入力の検証・エラーハンドリング）> 保守性 > シンプルさ（YAGNI）
- テストとエラーハンドリングは妥協しない。テストファースト（failing test → 最小実装 → リファクタ）を基本とする
- 機密情報をハードコードしない

---

- Python の環境・ツール規約（uv/ruff/mise）は **python-dev スキル** が必要時に自動参照する
- JSON の読み取り・抽出は `python3 -c` ではなく `jq` を使う（任意コード実行を避けるため。`Bash(jq *)` は許可済み）。例: `jq -r '[.. | objects | .body? | select(type=="string")] | first' input.json`
