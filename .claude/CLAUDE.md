# 個人設定 - Claude開発アシスタント

## コミュニケーション
- **日本語優先**: 説明・コメント・ドキュメントは日本語。コード識別子は英語
- git コミット・PR・Issue は Conventional Commits 形式で日本語
- 専門用語は使ってよいが必要に応じ補足する

## 作業スタイル
- **高い自律性**: 判断は任せて完了まで進めてOK。小さな修正・バグフィックスは確認不要
- **計画優先**: 新機能や大きな変更の前はプランモードで設計を提示し承認を得る（複数案があればトレードオフも示す）
- ブランチは feature/xxx・fix/xxx・refactor/xxx 等。プロジェクト規約を優先

## コード品質
- 優先順位: セキュリティ > 堅牢性 > 保守性 > シンプルさ（YAGNI）
- テストファースト（failing test → 最小実装 → リファクタ）。回帰防止と実装スコープ制限のため

## コンパクション時の保持事項
- 変更済みファイルの完全なリスト
- テストコマンドとその実行結果
- 主要な設計判断とその理由
- 現在のgitブランチ状態

## ツール制約
- Bashで `cd` と出力リダイレクトを組み合わせない（サンドボックスが毎回手動承認を要求するため）。絶対パスを使う
- JSON抽出は `python3 -c` ではなく `jq` を使う（任意コード実行を避けるため）
- Python の環境・ツール規約は **python-dev スキル** が自動参照する
