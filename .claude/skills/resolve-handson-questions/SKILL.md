# Resolve Handson Questions

ハンズオン教材（複数章のディレクトリ構成）全体から `//` 疑問コメントを探し出し、一次ソースで裏取りした回答で置換する。ファイル指定は不要。

## ワークフロー

### Phase 1: 対象ハンズオンの特定と疑問の検出

1. 対象ディレクトリを決める:
   - 会話の文脈でハンズオンが特定できていればそのディレクトリ
   - 不明なら Obsidian vault（`~/Documents/obsidian_vault`）を候補ルートとして検出スクリプトを実行し、ヒットしたファイルの親ディレクトリから対象を判断する。複数のハンズオンにまたがってヒットした場合のみユーザーに確認する
2. 検出スクリプトを実行する:
   ```bash
   scripts/find-question-comments.sh <対象ディレクトリ>
   ```
   全 `.md` を再帰検索し、**フェンスコードブロック外**の `//` 行を `ファイル:行番号:内容` で列挙する（コードブロック内の `//` コメントと本文中の URL は自動除外済み）
3. 出力から疑問コメントだけを選別する（プロトコル相対 URL 等のまれな誤検知を目視で除外）。疑問表現がない `//` 行も、学習者のメモとして書かれていれば対象に含める
4. ファイル別の疑問一覧と件数をユーザーに報告してから調査に進む

### Phase 2: 一次ソース調査（必須）

回答は必ず一次ソースで裏取りする。二次ソース（ブログ・Qiita 等）だけを根拠に回答しない。

1. **教材内の出典を先に確認する**: ハンズオン各章の `> 参考:` リンクと付録の参考文献に一次ソースが列挙されていることが多い。まずそこから該当ページを特定する
2. **一次ソースの優先順位**: 公式ドキュメント（developers.cloudflare.com, docs.aws.amazon.com 等）> RFC・仕様書 > 公式ブログ・リリースノート。ライブラリ/クラウドサービスの仕様は Context7 MCP（resolve-library-id → query-docs）で最新ドキュメントを取得し、不足は WebFetch/WebSearch で補完する
3. 疑問が複数ある場合は Agent ツールで並行リサーチする（`run_in_background: true`、1エージェント1〜2疑問）。各エージェントには「事実 + 出典 URL をセットで返す」よう指示する
4. 一次ソースで確認できなかった事項は、回答内で「一次ソース未確認」と明示する

### Phase 3: 回答の構成と挿入

1. 回答は教材の文体・構成規約に合わせる:
   - `//` コメント行を `> **補足: <疑問の要約>**` で始まる引用ブロックに **Edit で置換**する（コメント行は残さない）
   - 回答の末尾に出典を `> 参考: [タイトル](URL)` 形式で必ず添える。変わりやすい事実（価格・プラン制限・レート制限）には時点を付記する
   - 比較を含む回答はテーブル、コード例は既存コードブロックのスタイルに揃える
   - その章の学習目標・既出概念と関連づけて説明し、教材の一部として自然に読めるようにする
2. 調査で「教材本文の記述が不正確」と判明した場合は、本文を勝手に書き換えず、補足内で正確な情報と出典を提示する
3. 独立した箇所は並行で Edit する。疑問が 10 件を超える場合は TaskCreate で進捗管理する

### Phase 4: 検証と報告

1. 検出スクリプトを再実行し、疑問コメントの残存がゼロであることを確認する
2. 対応結果をファイル別の一覧表（章・疑問・回答要旨・出典）でユーザーに報告する

## Resources

### scripts/find-question-comments.sh

対象ディレクトリ以下の全 Markdown からフェンスコードブロック外の `//` 行を列挙する検出スクリプト。引数はディレクトリ 1 つ。出力は `ファイルパス:行番号:行内容`、ヒットなしなら出力なしで exit 0。
